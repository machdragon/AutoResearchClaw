# ZQAT: Codec-Roundtrip QAT Infrastructure for Parameter-Golf Lane 3

# Abstract

Parameter Golf lane 3 poses a tightly constrained compression setting: models must be deployed as int8 weights compressed with zlib under a 16 MB artifact cap and a ≤1.10× runtime quick-gate, with validation bits-per-byte (val_bpb) measured only after a full quantization–codec roundtrip. Standard quantization-aware training (QAT) and post-training quantization (PTQ) optimize float-space or fake-quantized objectives and interact with deployment codecs only at export time, leaving a gap between training objectives and the true leaderboard metric. This work introduces ZQAT, a codec-roundtrip QAT framework that embeds the int8+zlib deployment stack into a staged training schedule tailored to Parameter Golf lane 3. ZQAT formalizes val_bpb on the post-roundtrip model, defines a gate-budgeted schedule of codec calls, and specifies an ablation suite spanning Float+PTQ, entropy-shaped PTQ (LayoutQ), staged QAT without codec-in-loop (StQAT), and full codec-aligned ZQAT. The current implementation integrates quantization, serialization, and zlib-based roundtrip evaluation into the official lane_3 harness but has not yet produced any successful training runs or recorded metrics, so no empirical improvements or numerical comparisons are reported. Instead, we present the design, implementation status, and planned evaluation protocol for testing whether staged, codec-aligned QAT can reduce post-roundtrip val_bpb under the same artifact and runtime gates as float-only and codec-agnostic baselines.

# Introduction

Compression-oriented deployment of neural sequence models demands optimization procedures that treat storage and codec behavior as first-class concerns alongside predictive performance. In Parameter Golf lane 3, this requirement is explicit: models must be delivered as zlib-compressed int8 artifacts no larger than 16,000,000 bytes, evaluated under a quick-gate that limits end-to-end runtime to at most 1.10× a standardized baseline, and ranked by validation bits-per-byte (val_bpb) computed after a full quantization–compression–decompression roundtrip. Under these constraints, training a high-quality float model and applying off-the-shelf post-training quantization (PTQ) or quantization-aware training (QAT) is not sufficient, because the benchmark’s primary metric depends on how int8 weights interact with a fixed, non-differentiable codec rather than on float-space loss alone. This setting crystallizes a broader class of deployment scenarios where models function as compression engines, and compressibility under a locked codec is more critical than conventional accuracy.

Existing quantization and compression methods provide powerful tools for reducing precision and footprint but typically stop short of aligning with a specific deployment codec. Surveys of transformer compression and quantization report strong results for PTQ and QAT techniques that match float and low-precision predictions while optimizing task loss or accuracy [tang2024survey, hasan2024optimizing, huang2023efficient, chen2025scaling].
