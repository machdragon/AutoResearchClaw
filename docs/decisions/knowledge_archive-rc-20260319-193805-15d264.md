# Retrospective Archive: lane_3 – QAT + roundtrip‑aware compression alignment

Status: **REFINE** (pipeline shakeout; no empirical results yet)  
Scope: Parameter Golf lane_3, “staged QAT aligned to int8+zlib roundtrip” under:
- Artifact limit: **≤16 MB**
- Runtime quick-gate: **≤1.10× baseline**
- Metric: **post‑roundtrip val_bpb**

---

## 1. What was attempted

### 1.1 High‑level aim

Hypothesis:

> If we run **staged QAT** that is explicitly aligned to the **int8+zlib deployment roundtrip**, then **post‑roundtrip val_bpb** will improve vs **float‑only training + post‑hoc quantization**, under:
> - The same **gate budget** for compression/codec calls, and
> - The same **runtime quick‑gate** and **16 MB artifact** constraints.

The key difference vs standard QAT:

- Standard QAT: optimize task loss or float‐space val_bpb with fake quant; codec is applied only at export time.
- Target: optimize **for the actual deployed model**:
  - Float weights → int8 quantization → serialized bytes → **zlib** → stored artifact
  - At eval: decompress → int8 weights → inference
  - Metric: **val_bpb after this full roundtrip**.

### 1.2 Implemented components (design vs reality)

Implemented *conceptually*:

- A **codec‑aware path** in the lane_3 harness:
  - Float weights → int8 quantization
  - Serialization into a deterministic byte layout
  - zlib compression and decompression
  - Deserialization back to int8 tensors
  - Inference with int8 weights to produce val_bpb
- A **per‑seed runner** with a time/seed **quick‑gate** to enforce runtime caps:
  - Budget‑aware selection of how many seeds to run.
  - Infrastructure logs warnings like `SEED_WARNING` when seeds are reduced.
- A plan for **staged QAT** with “codec‑in‑loop”:
  - Early float training.
  - Intermediate fake‑quant QAT without codec calls.
  - Final stage where some steps include a full int8+zlib roundtrip to obtain an aligned signal.

Not implemented to completion (or not yet exercised):

- Stable QAT schedule integrated with the codec path.
- Reliable, repeated metric logging across seeds and configs.
- Baseline grid and ablations (see Section 3).

---

## 2. What actually happened in the run

### 2.1 Run outcome

- The **first end‑to‑end run failed** inside project code:
  - Crash occurred before any metric logging.
  - No `val_bpb`, no artifact size, no timing curves were written.
- Planned:
  - 3 seeds.
- Actual:
  - **0 seeds completed successfully**.
- The harness did:
  - Produce a `SEED_WARNING` about using only 3 seeds due to time budget.
  - Demonstrate that the **gating logic is active** and budget‑aware.

Net effect: we exercised the *outer shell* (orchestration, gating) but **never reached a complete codec‑roundtrip evaluation** for any seed.

### 2.2 Missing data

For every experimental condition, the following are missing:

- Post‑roundtrip **val_bpb**.
- **Compressed artifact size** (zlib bytes).
- **Runtime** (per seed/config).
- Any comparison between:
  - Float+PTQ+zlib vs entropy‑shaped formats,
  - PTQ vs QAT,
  - QAT vs codec‑in‑loop QAT.

There is **no empirical evidence** yet about whether codec‑aligned QAT helps or hurts.

---

## 3. Methodology design vs current implementation

### 3.1 Intended experimental grid

Targeted configurations (not yet fully wired):

1. **B0 – Float + Naive PTQ + zlib**
   - Train in float32 only.
   - At the end: per‑tensor or per‑channel INT8 PTQ.
   - Serialize weights, compress with zlib, then decompress and evaluate.
   - Purpose: “plain” baseline.

2. **B1 – Float + Entropy‑shaped INT8 layout + PTQ/light QAT + zlib (LayoutQ)**
   - Same float training budget as B0.
   - Better weight layout and packing to expose redundancy to zlib.
   - Either PTQ or very light QAT to preserve deployment-friendly distributions.
   - Purpose: isolate format/layout contributions to compressibility.

3. **B2 – Staged QAT without codec‑in‑loop**
   - Same layout and INT8 scheme as B1.
   - Staged QAT (fake quantization in training) but **no codec calls** in the loop.
   - Final export/eval uses int8+zlib, but training only “sees” quantization, not zlib.
   - Purpose: isolate QAT effect vs PTQ under identical deployment pipeline.

4. **E0 – ZQAT: Staged QAT with codec‑in‑loop**
   - Same quantization and layout as B2.
   - During later training, occasionally:
     - Run full float → int8 → serialize → zlib → de‑zlib → de‑serialize → int8 inference
     - Use that signal (e.g., val_bpb or a proxy) to adjust weights.
   - Subject to a **gate budget** on the number of codec calls to respect runtime.

Planned commonalities:

- Same **dataset and tokenization** (via Parameter Golf lane_3 API).
- Same **train/val/test splits**.
- Same total **training steps/epochs** and optimizer (within reason).
- Identical INT8 scheme (scales, zero‑points) for conditions that compare directly.
