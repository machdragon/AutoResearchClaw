## Cluster 1 – Complexity vs Performance Under Hard Budgets

### Key Analogues
- **Zhang 2020 (multi‑size + triplet loss + dropout)**: improves accuracy via training‑only additions with *no* inference‑time parameter or runtime cost.
- **Hong 2020 (RF vs OLS)**: nonlinear, higher‑capacity RF dramatically reduces error compared with linear OLS under the same feature budget.
- **Zhou 2023 (Au/Cu/Ag wires)** and **Trzepieciński 2024 (automotive metals)**: systematic evaluation under multi‑dimensional constraints (cost, reliability/performance, manufacturability).
- **Woody 2024 (EV TCO)**: parametric modeling of tradeoffs across locations/conditions.

### Synthesis for Lane_1

1. **Training‑only vs inference‑time modifications**
   - Zhang’s pattern suggests: exhaust *training‑time* tricks before spending inference budget.
   - For lane_1, that argues for:
     - Recurrent backbone kept fixed in parameter count and unrolled steps.
     - LoRA and gating modules designed so that at inference some parts can be *folded* or sparsified.
     - Heavy auxiliary losses or auxiliary recurrent paths only used during training and pruned for deployment.

2. **Nonlinearity/specialization vs linear baseline**
   - Hong’s OLS→RF step is analogous to:
     - Simple recurrence‑only model ≈ “OLS”
     - Recurrence + LoRA specialization + gating ≈ “RF”: same inputs, but piecewise or conditional capacity.
   - The magnitude of improvement in Hong (≈4× reduction in error) sets the expectation: *if gating actually routes tokens/states to specialized LoRA adapters under similar parameter count*, val_bpb could show a meaningful drop, not just marginal wins.

3. **Multi‑objective evaluation framework**
   - Materials and EV TCO reviews emphasize you must:
     - Enumerate constraints explicitly.
     - Evaluate configurations on a common, multi‑axis score.
   - Translated:
     - Axes: `val_bpb`, runtime multiplier, artifact size (≤16MB), training cost.
     - Systematic sweep of LoRA rank, number of experts/gates, and recurrent depth to populate this Pareto surface.

4. **Context‑sensitivity and “location‑based” cost**
   - EV TCO shows that cost‑competitiveness is strongly context dependent.
   - For lane_1, context = *data distribution and sequence length distribution*:
     - A configuration that is runtime‑neutral on average may breach 1.10× on long‑sequence tails if gating adds overhead per time‑step.
     - Need stratified runtime measurement: short vs long sequences, low vs high routing entropy.
