# Stage 01: Topic Init

- **Topic**:  
  Parameter Golf val_bpb minimization — lane_6: Byte-grouped serialization for artifact compression in quantized language models

---

### Novel Angle

Most quantization and model-compression work optimizes **runtime metrics or perplexity** under a fixed model-serialization format (e.g., flat arrays, tensor-wise bit-packing, or standard ONNX/gguf-esque layouts). In contrast, this project targets the **joint design of quantization + serialization layout** specifically to:

> Minimize *artifact bytes* (the on-disk weight file) via **byte-grouped packing** while keeping **post-roundtrip val_bpb** and **runtime** effectively unchanged.

What is new here?

1. **Compression objective is val_bpb, not perplexity / loss only**  
   - Existing works mostly optimize for perplexity or task accuracy. Parameter Golf’s *val_bpb* (bits-per-byte on validation data) is a **rate-like, compression-flavored** metric specific to language modeling pipelines that care about tokenization and coding efficiency.  
   - Optimizing serialization to reduce **artifact size** under a **val_bpb constraint** is not standard in quantization literature, which usually measures quality as loss, perplexity, or BLEU.

2. **Serialization layout as a first-class optimization variable**  
   - Quantization papers typically treat serialization as an implementation detail: weights are stored row-major, tensor-major, or block-wise for cache friendliness, but the **byte-level layout is not co-designed** with the quantization scheme for artifact-size minimization under **roundtrip constraints**.  
   - Here, we explicitly test **byte-grouped packing** (e.g., grouping weights such that bytes of equal significance or similar local statistics are contiguous) while holding quantization parameters constant.  
   - The hypothesis is that more “entropy-homogeneous” byte neighborhoods can be compressed better (e.g., via downstream artifact compressors) without harming the model after deserialization.

3. **Explicit tradeoff among (artifact size, val_bpb, runtime)** with a **hard runtime quick-gate**  
   - Many compression works mention runtime overhead qualitatively; few enforce a strict **runtime budget** (≤ 1.10× baseline) as a *gating* condition for accepting a layout change.  
   - This is closer to a compiler/ML-systems–style constraint but applied to **quantized LLM weight artifacts** and evaluated via **val_bpb** instead of conventional latency-only metrics.

4. **Timeliness (2024–2026)**  
   Recent trends make this particularly relevant now:
   - Open-weight LLMs and fine-tuning frameworks (LoRA, QLoRA, AWQ, GPTQ, gguf ecosystems) place strong emphasis on **redistributable artifacts** (HuggingFace, custom hubs) where **file size and download time** are critical.  
   - Interest in **low-resource deployment** (edge, mobile, browser) is exploding, where artifact size ceilings (e.g., 16 MB here) and runtime tightness are strong constraints.  
   - New work on **codecs and entropy-aware layouts** for model weights (e.g., product quantization, residual quantization, weight clustering) indicates that structure in the weight distribution is exploitable—but existing work rarely studies **byte-level packing patterns** as a separate design dimension under strict runtime bounds.

5. **Distinct from standard approaches**  
   - Not doing “another quantization method” or “another compression codec” benchmark.  
   - Focus is on **serialization layout design** for already-quantized weights:  
     - Controlled experiment: *same* quantizer, *same* model, *same* number of quantization bits.  
     - Only change: **how quantized integers are arranged and packed into bytes** to reduce artifact size while checking post-roundtrip val_bpb and runtime.  
   - This is underexplored because most ML papers assume that changing file layout is an implementation micro-optimization, not a publishable research axis—yet with modern scale and bandwidth costs, this axis matters.
