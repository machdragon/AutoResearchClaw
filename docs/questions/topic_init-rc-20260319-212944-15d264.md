# Stage 01: Topic Init

- **Topic**: Parameter Golf val_bpb minimization for compression-aware QAT (lane_3: QAT + roundtrip-aware compression alignment)

---

### Novel Angle

**Core idea**  
Study *staged quantization-aware training (QAT)* where the optimization objective is explicitly aligned to a full **int8 → zlib → int8 roundtrip** under:

- a **strict artifact size limit** (≤ 16 MB for the full model artifact), and  
- a **runtime quick-gate constraint** (≤ 1.10× baseline FLOPs/runtime),  

with the evaluation metric being **post-roundtrip validation bits-per-byte (val_bpb)** on a language modeling task.

The hypothesis:  
> If we enable staged QAT aligned to an actual int8+zlib roundtrip, then *post-roundtrip* val_bpb will improve versus a strong float-only baseline trained under the same “gate budget” (runtime + size) constraints.

**Why this is not already covered**

1. **Most QAT work optimizes pre-quantization loss, not post-roundtrip code-length**  
   - Standard QAT (e.g., for LLMs and vision models) focuses on minimizing the loss (or perplexity) of the *quantized* model, assuming a simple quantization operator (int8, int4, etc.), but rarely models the **effect of further entropy compression (e.g., zlib)** on:
     - file-size of the final artifact, and  
     - predictive performance *after* a real-world serialization–deserialization roundtrip.
   - Existing compression-aware training often considers **rate–distortion** or **entropy regularization** for weights or activations, but **not** the combined, practical pipeline **float → int8 weight pack → zlib compress → store → zlib decompress → reload → run**.

2. **Roundtrip-aware alignment for LM *bits-per-byte* is underexplored**  
   - For language models, leading work usually optimizes **token-level cross-entropy / perplexity**, not **val_bpb**, under an explicit **compression pipeline**.
   - Parameter Golf–style exploration of *val_bpb* is emerging (treating LM as a compressor), but most work:
     - tunes architectures / hyperparameters in float,
     - then applies quantization or model compression post hoc.
   - There is little systematic analysis of **training-time adaptation specifically to “int8+zlib” artifacts**, where the goal is *not* just low val_loss but explicitly **better compressed model artifact size vs. predictive bits-per-byte tradeoff**.

3. **Joint constraints: gate budget + artifact limit + val_bpb**  
   - Many model compression papers assume:
     - unconstrained or loosely constrained train-time compute, or  
     - primarily focus on runtime/throughput or model size in isolation.
   - This project explicitly couples:
     - **Gate budget**: QAT schedule must keep runtime ≤ 1.10× a baseline float-only run.
     - **Artifact limit**: final full artifact ≤ 16 MB (post int8 packing + zlib).
     - **Objective**: **post-roundtrip val_bpb**, not just validation loss.
   - This *triple constraint* (runtime + size + compression-roundtrip metric) is unusual and matches real-world deployment regimes (e.g., on-device, bandwidth-constrained models).

4. **Staged QAT with compression-aware curriculum**  
   - Typical QAT either:
     - quantizes from the start, or
     - applies a simple, monotonic annealing of fake-quant noise.
   - Here we propose **staged QAT that is explicitly aligned to when and how int8 packing + zlib will be applied**, e.g.:
     - Stage 1: float training, but add regularizers that encourage “zlib-friendly” weight distributions.  
     - Stage 2: mixed-precision / fake-quant training where channel-wise scaling is chosen to reduce *post-zlib* entropy of packed weight tensors.  
     - Stage 3: fully quantized int8 training with periodic simulated **full roundtrips** (quantize → pack → zlib → decompress → unpack) baked into training steps or checkpoints.
   - This “roundtrip-aligned curriculum” as a *first-class training schedule* targeting val_bpb is novel relative to conventional QAT workflows.

**Why this is timely (2024–2026)**

- **On-device / edge LM deployment** is surging; models must be:
  - small (artifact limits: phones, microcontrollers, browsers),
  - fast (strict runtime budgets),
  - *good compressors* (val_bpb matters for both internal compression tasks and energy/bandwidth use).
- **Recent trends**:
  1. **Ultra-low-bit LLMs and QAT** (2023–2025) show strong interest in making LLMs run at 4–8 bits with minimum quality loss.
  2. **Compression-aware training frameworks** and **bits-back / information-theoretic** views of LMs are gaining traction, but rarely integrated with practical storage codecs like zlib.
  3. **Parameter Golf / auto-rc style projects** highlight the value of aggressively exploring the design space under strict runtime and size constraints; however, most stop at model architecture & training hyperparameters, not at the **file-compression pipeline itself**.
