# Stage 08: Hypothesis Gen

Here is a synthesized set of four hypotheses that combine the most novel ideas, respect feasibility constraints, and surface the contrarian’s core challenges rather than smoothing them away.

---

## Hypothesis 1 – Minimal “Self‑Organizing Codebook” QAT Beats Strong Post‑Hoc Search at Fixed Budget

**Core idea**

A *very short*, entropy‑regularized QAT “critical period” that directly shapes int8 weights can produce zlib‑friendlier codebooks that **post‑hoc quant/codec search cannot reach** under the same total budget—even when the post‑hoc pipeline is allowed to use the *entire* 10% extra budget that QAT consumes.

### Rationale

- From the innovator:
  - There is a cheap, *online* handle on zlib‑relevant structure: local int8 deltas and their entropy.
  - A brief QAT window where we nudge distributions while gradients still flow may move the model into an attractor that is hard to discover via purely offline search.
- From the pragmatist:
  - We can implement this in a minimal way: fake‑quant on a few big layers, one small histogram per step, a tiny extra loss term; within ≤1.10× runtime.
  - QAT is partial: only scales/adapters for select layers.
- From the contrarian:
  - Post‑hoc search over layouts, groupings, per‑layer quant schemes may be quite powerful.
  - To be meaningful, QAT must beat *that* baseline, not a naïve PTQ pipeline.

This hypothesis therefore makes a **head‑to‑head claim**: even granting the contrarian a strong post‑hoc search, there is still additional compressibility that only in‑loop QAT can unlock.

### Measurable prediction

Consider three systems trained from the same float checkpoint under the same base token budget:

1. **Float‑only baseline**  
   - Float training for T steps.  
   - Then naïve int8 PTQ + fixed layout + zlib.  
   - Metrics: `val_bpb_baseline`, `size_baseline`.

2. **Best‑effort post‑hoc system (contrarian’s champion)**  
   - Same T float steps; **no QAT**.  
   - Use up to +10% extra wall‑clock on:
     - searching over per‑layer / per‑group quantization schemes (int8 only),
     - packing/layout permutations (e.g., channel reordering, block layouts),
     - a *very short* adapter‑only finetune after quant, if desired.  
   - Always evaluate with *real* int8+zlib.  
   - Take the best configuration over this search: `val_bpb_posthoc`, `size_posthoc`, `time_posthoc`.

3. **Minimal self‑organizing QAT (proposed)**  
   - Same T steps, but:
     - Last `T_q` steps (e.g., 5–10% of T) use staged QAT on the largest M layers:
       - fake‑quant of weights (and optionally activations),
       - trainable int8 scales and small adapters.
     - On each QAT step:
       - collect flattened int8 weights for those layers,
       - compute delta stream Δ and a clipped histogram,
       - entropy proxy `H_Δ`,
       - add `L_entropy = λ H_Δ` with λ tuned so `L_entropy ≈ 1–2%` of task CE.  
   - No zlib calls in the loop; only at the very end.  
   - Same final int8 scheme and layout as the post‑hoc system (to isolate the effect of QAT).  
   - Metrics: `val_bpb_QAT`, `size_QAT`, `time_QAT`.

**Prediction 1.1**

Under the constraint `time_QAT ≤ 1.10 × time_baseline` and `time_posthoc ≤ 1.10 × time_baseline`:

- `val_loss_QAT ≤ 1.002 × val_loss_baseline` (≤0.2% relative perplexity degradation).  
- `val_bpb_QAT ≤ 0.97 × val_bpb_baseline` (≥3% bpb improvement vs naïve).  
- Crucially:  
  `val_bpb_QAT ≤ val_bpb_posthoc − 0.5%` (QAT beats *best* post‑hoc by at least 0.5% relative bpb),  
  with `size_QAT ≤ size_posthoc` (no artefact‑size regression).

### Failure condition

Hypothesis 1 is rejected if, under matched constraints:

- `time_QAT > 1.10 × time_baseline`, **or**
- `val_loss_QAT > 1.002 × val_loss_baseline`, **or**
- QAT does not produce a clear compression win:
  - `val_bpb_QAT ≥ 0.99 × val_bpb_baseline` **or**
  - `val_bpb_QAT > val_bpb_posthoc − 0.005 × val_bpb_posthoc` (QAT fails to beat the best post‑hoc by at least 0.5% relative).

### Unresolved disagreements this tests

- **Innovator/Pragmatist vs Contrarian (Alt A):**  
  Is there genuinely new zlib‑relevant structure that only training‑time QAT can reach, or is good post‑hoc engineering enough at int8?  
  This hypothesis sides with QAT but makes the comparison explicit and fair.

## Hypothesis 2 – Sparse “Codec Curriculum” with Real Zlib Probes Outperforms Proxy‑Only or No‑Codec QAT

1. a cheap proxy (delta entropy + simple run‑length score) for most steps, and

- QAT with only proxy losses, and
